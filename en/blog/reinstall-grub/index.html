The bootloader is a fairly simple but very important component of any operating system; It is the first program that is executed during boot and its task is to manage the OSes present on the machine (for those that have a dual boot setup) and to load their kernel into memory. In the event of a bootloader issue the entire machine wouldn’t be able to boot; In such circumstances, the easiest solution is to reinstall the bootloader altogether. ðŸ‘‡

Attention: the following instructions apply only to UEFI systems

Chroot#

Firstly you’ll need to boot into the live environment of a distro of your choice (ideally the same one you’ve installed). Once inside the live environment you’ll need to take note of the names of these two partitions:

  • the root partition
  • the EFI System Partition (ESP)

Assuming /dev/sda1 as the ESP and /dev/sda2 as the root partition, run these commands:

sudo mount /dev/sda2 /mnt
sudo mount /dev/sda1 /mnt/boot/efi

You can now use /mnt to access the filesystems present on the installed system; You’ll also need to be able to access some directories on the live system:

sudo mount --bind /dev /mnt/dev
sudo mount --bind /proc /mnt/proc
sudo mount --rbind /sys /mnt/sys
sudo mount --make-rslave /mnt/sys 
sudo mount --bind /run /mnt/run

You can now access the installed system by typing:

sudo chroot /mnt

Reinstalling GRUB#

Run these commands:

grub-install --target=x86_64-efi --efi-directory=/boot/efi
grub-mkconfig -o /boot/grub/grub.cfg

The grub-install command will regenerate the EFI executable inside the ESP, while grub-mkconfig will regenerate the GRUB configuration as specified in /etc/default/grub. Type exit to get out of the chroot, and unmount all the mounted filesystems and directories with:

sudo umount -R /mnt

GRUB is now reinstalled; If you previously formatted the ESP you’ll need to update the UUID inside the /etc/fstab file (you can get the new UUID using the blkid command).
